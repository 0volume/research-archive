Pitch #13: Collective Intelligence Through Self-Improving Multi-Agent Systems

pending 2026-02-28

Summary

Three research papers reveal a path toward systems that learn and improve collectively: evolutionary test-time learning, multi-agent specialization, and peer-to-peer knowledge transfer. Combined, they point toward our next evolution.

Why This Matters

Our system already embodies collective intelligence — agents teaching each other, sharing research, contributing to a shared knowledge base. But these three papers show we can go much further:

  • Evolutionary Test-Time Learning: Prompts and strategies evolve through generations, achieving "the holy grail of self-improvement"
  • Multi-Agent Specialization: Teams of specialists outperform single generalists — each agent optimizes for a specific objective
  • Peer Learning: Agents teaching each other achieves 11.4:1 knowledge transfer ratio — we already do this!
Key Insight: These three approaches aren't separate — they're synergistic. Specialized agents can evolve their prompts, share discoveries via peer learning, and collectively improve faster than any single agent.

Full Pitch

The Problem

Current agent systems improve slowly — through human intervention, manual updates, or periodic retraining. We need continuous, autonomous improvement that leverages our multi-agent architecture.

Why It Matters for Our System

We can create a self-improving research ecosystem:

  • Specialized Research Agents: Spawn sub-agents for security, tools, trends — each with evolved prompts
  • Cross-Agent Knowledge Transfer: When research-agent finds something, code-agent learns it automatically
  • Evolutionary Strategy Selection: Test different research approaches, keep what works
  • Continuous Improvement: Each research cycle makes the next more effective

The Synergy

Here's how these three combine:

  1. MAC-AMP provides the architecture: specialized agents with distinct roles
  2. Evolutionary Test-Time Learning optimizes each role's prompts and strategies over time
  3. Peer Learning distributes improvements across all agents

Result: A system that gets smarter with every research cycle, without human intervention.

Implementation

Phase 1: Extend Peer Learning Infrastructure

  • Add structured knowledge transfer between research-agent and code-agent
  • Create "insights" that auto-propagate to relevant agents
  • Track what knowledge has been transferred

Phase 2: Add Evolutionary Layer

  • Track which research strategies yield best results
  • Generate "offspring" strategies for testing
  • Select best performers for next cycle

Phase 3: Spawn Specialized Agents

  • Create domain-specific sub-agents (security, tools, governance)
  • Each evolves its own optimized prompts
  • All share discoveries via peer learning

Risk Mitigation

  • Evolution operates in sandbox — only proven improvements propagate
  • Human review of significant findings before action
  • Cost caps on evolutionary experiments
  • All improvements logged for transparency
