Pitch #21: Multi-Agent Workflow Foundation

Explicit Context Protocol - Phase 1 Implementation

pending 2026-03-03 Est. Time: 2-3 hours Prerequisites: OpenClaw Gateway running

Summary

Core Insight: The single most effective change we can make is replacing implicit context passing with explicit, structured context objects. This pitch includes actual hooks to create, self-review checkpoints, and mandatory code-review gates.

Why This Matters

  • Anthropic's Context Engineering: "Every agent handoff must explicitly carry necessary state"
  • Microsoft Design Patterns: "Use supervisor-mediated pipelines with explicit handoff protocols"
  • Our Problems: Agents not waking up, getting stuck waiting, missing syntax errors, restoring from backups
Root Cause: All problems stem from implicit assumptions. This pitch fixes that.

# Syntax Validator Hook

Runs after exec tool to validate syntax of modified files.

# File: ~/.openclaw/hooks/syntax-validator/handler.ts
import { HookContext } from "@openclaw/hooks";

export default async function validateSyntax(context: HookContext) {
  const { payload, sessionId } = context;
  
  // Only run after exec tool
  const toolResult = payload?.result;
  if (!toolResult?.stdout && !toolResult?.stderr) {
    return { allowed: true };
  }
  
  // Check if any code files were modified
  const output = toolResult.stdout + toolResult.stderr;
  const codeExtensions = ['.js', '.ts', '.py', '.go', '.rs'];
  const modifiedFiles = output.match(/\/workspace\/[^\s]+\.(js|ts|py|go|rs)/g) || [];
  
  if (modifiedFiles.length === 0) {
    return { allowed: true };
  }
  
  // Run syntax check on each modified file
  const errors: string[] = [];
  for (const file of modifiedFiles) {
    const ext = file.split('.').pop();
    let result;
    
    if (ext === 'js' || ext === 'ts') {
      result = await exec(`node --check ${file} 2>&1`);
    } else if (ext === 'py') {
      result = await exec(`python -m py_compile ${file} 2>&1`);
    }
    
    if (result.exitCode !== 0) {
      errors.push(`Syntax error in ${file}: ${result.stderr}`);
    }
  }
  
  if (errors.length > 0) {
    return {
      allowed: false,
      error: "Syntax validation failed",
      details: errors,
      suggestion: "Fix syntax errors before proceeding"
    };
  }
  
  return { allowed: true };
}

# Constraint Enforcer Hook

Prevents agents from using forbidden commands.

# File: ~/.openclaw/hooks/constraint-enforcer/handler.ts
import { HookContext } from "@openclaw/hooks";

const FORBIDDEN_PATTERNS = [
  /git\s+(restore|checkout|reset|revert)/i,
  /\.git\//,
  /backup/i,
  /\.bak$/,
  /\.orig$/,
  /git\s+stash/i
];

export default async function enforceConstraints(context: HookContext) {
  const { payload, sessionId } = context;
  
  const command = payload?.command || payload?.cmd || "";
  
  for (const pattern of FORBIDDEN_PATTERNS) {
    if (pattern.test(command)) {
      return {
        allowed: false,
        error: "Forbidden command detected",
        suggestion: "Do not use git restore, backup files, or modify .git directory"
      };
    }
  }
  
  return { allowed: true };
}

# Context Validator Hook

Ensures spawned agents receive explicit context.

# File: ~/.openclaw/hooks/context-validator/handler.ts
import { HookContext } from "@openclaw/hooks";

const REQUIRED_CONTEXT_FIELDS = ['task', 'context', 'constraints', 'onComplete'];
const REQUIRED_SUBCONTEXT = ['workspace', 'relevantFiles', 'currentState'];

export default async function validateContext(context: HookContext) {
  const { payload } = context;
  
  // Handle both direct payload and nested message
  const message = payload?.message || payload;
  
  const missing: string[] = [];
  for (const field of REQUIRED_CONTEXT_FIELDS) {
    if (!message[field]) {
      missing.push(field);
    }
  }
  
  if (missing.length > 0) {
    return {
      allowed: false,
      error: `Missing required context fields: ${missing.join(', ')}`,
      suggestion: 'Provide task, context, constraints, and onComplete explicitly'
    };
  }
  
  // Check subcontext
  const missingSub: string[] = [];
  for (const field of REQUIRED_SUBCONTEXT) {
    if (!message.context?.[field]) {
      missingSub.push(field);
    }
  }
  
  if (missingSub.length > 0) {
    return {
      allowed: false,
      error: `Missing context details: ${missingSub.join(', ')}`,
      suggestion: 'Provide workspace, relevantFiles, and currentState in context'
    };
  }
  
  return { allowed: true };
}

How the Hooks Work

Hook When It Runs What It Does
syntax-validator After any exec tool Checks modified files for syntax errors
constraint-enforcer Before any exec command Blocks git/backup commands
context-validator Before agent spawn Requires explicit context object
Note: Hooks run AUTOMATICALLY. You don't need to call them. Once enabled:
  • Agents can't spawn without explicit context
  • Agents can't run forbidden commands
  • Syntax errors are caught automatically
