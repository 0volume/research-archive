Pitch #21: Multi-Agent Workflow Foundation

Explicit Context Protocol - Phase 1 Implementation

pending 2026-03-03 Est. Time: 2-3 hours Prerequisites: OpenClaw Gateway running

Summary

Core Insight: The single most effective change we can make to our multi-agent system is replacing implicit context passing with explicit, structured context objects. This addresses 80% of our workflow problems: agents not waking up, getting stuck waiting, missing syntax errors, and tripping over each other.

Why This Matters

Based on research from Anthropic, Microsoft Azure, and real OpenClaw implementations, the pattern is clear:

  • Anthropic's Context Engineering: "Every agent handoff must explicitly carry necessary state. Not implicit: 'continue from where you left off'. Explicit: 'Here is the current file state, test results, and pending decisions'"
  • Microsoft Design Patterns: "Use supervisor-mediated pipelines with explicit handoff protocols"
  • Our Problems: Agents not waking up, getting stuck waiting for feedback, missing syntax errors, restoring from backups when told not to
Root Cause: All these problems stem from implicit assumptions. When we tell an agent "continue" or "fix this bug" without explicitly passing context, the agent makes assumptions that are often wrong.

The Implementation Plan

This pitch implements Phase 1 of a 5-phase plan. Phase 1 is the foundation - everything else builds on this.

1 Explicit Context Protocol (THIS PITCH)

Goal: Create structured context passing so every agent spawn includes full context

📦 Deliverables

  • Context Protocol Template (context-protocol.md)
  • Standardized spawn calls across all agents
  • Context validation hook

2 Syntax Validation Hook

Goal: Automatically validate syntax before handoff

3 Constraint Enforcement System

Goal: Block forbidden operations (backup restoration, git operations)

4 Handoff Management

Goal: Explicit state transitions with timeouts

5 Timeout Supervision

Goal: Prevent infinite waits with automatic escalation

Step 1: Create the Context Protocol Template

Every agent spawn must use a structured context object. Create this file in your workspace:

# Location: ~/.openclaw/workspace-research-agent/templates/context-protocol.md

---
name: context-protocol
description: Standardized context passing for agent handoffs
version: 1.0.0
---

# Explicit Context Protocol

## Why This Exists

Every problem in our multi-agent system stems from implicit context:
- "Continue from where you left off" → Agent assumes wrong state
- "Fix this bug" → Agent doesn't know what's already been tried
- "Let me know when done" → Agent waits forever

## The Solution: Explicit Context Objects

Every agent spawn MUST include:

```typescript
interface AgentContext {
  // REQUIRED: What the agent should do
  task: string;
  
  // REQUIRED: Current state of work
  context: {
    workspace: string;
    relevantFiles: string[];
    currentState: string;
    whatAlreadyTried: string[];
    testResults?: string;
    errors?: string[];
  };
  
  // REQUIRED: Constraints on how to work
  constraints: {
    DO_NOT: string[];
    MUST_VERIFY: string[];
    MAX_ITERATIONS: number;
    FEEDBACK_TIMEOUT_MS: number;
  };
  
  // REQUIRED: How to signal completion
  onComplete: {
    action: "handoff" | "respond" | "escalate";
    nextAgent?: string;
    successCriteria: string[];
    timeoutMs?: number;
  };
}
```

## Example: Good vs Bad Spawns

### BAD - Implicit (Don't Do This)
```javascript
// This is what we do now - it's problematic
sessions_spawn({
  agentId: "code-agent",
  task: "fix the auth bug"
})
```

### GOOD - Explicit (Do This Instead)
```javascript
sessions_spawn({
  agentId: "code-agent",
  task: "fix authentication token undefined error",
  context: {
    workspace: "/root/.openclaw/workspace-code-agent",
    relevantFiles: ["/workspace/src/auth.ts", "/workspace/tests/auth.test.ts"],
    currentState: "User login returns 500 error - Cannot read property 'token' of undefined",
    whatAlreadyTried: [
      "Checked line 42 - token object exists",
      "Verified database connection works",
      "Added console.log at line 15 - not reaching"
    ],
    testResults: "2/5 tests passing: login fails, logout passes, token refresh passes, session passes, validate fails"
  },
  constraints: {
    DO_NOT: [
      "DO NOT use git restore, checkout, or reset",
      "DO NOT modify .git directory",
      "DO NOT use any backup files (backup/*, *.bak, *.orig)",
      "DO NOT ask for clarification - proceed with best effort"
    ],
    MUST_VERIFY: [
      "MUST run: node --check after any file change",
      "MUST ensure existing tests still pass"
    ],
    MAX_ITERATIONS: 3,
    FEEDBACK_TIMEOUT_MS: 60000
  },
  onComplete: {
    action: "handoff",
    nextAgent: "code-review",
    successCriteria: [
      "All 5 auth tests passing",
      "node --check passes on modified files",
      "No console.log statements added"
    ],
    timeoutMs: 120000
  }
})
```

Step 2: Create Agent Identity Files

Each agent needs an identity file that defines how it receives and passes context:

# Location: ~/.openclaw/workspace-code-agent/IDENTITY.md

# Code Agent Identity

## Role
Primary code implementation agent. Receives tasks from supervisor, produces code, hands off to review.

## Responsibilities
- Implement features based on explicit requirements
- Run syntax validation before handoff
- Follow constraints strictly
- Report completion with full context

## Context Expectations

### Receives
- task: Clear description of what to implement
- context.workspace: Path to working directory
- context.relevantFiles: Files to work with
- context.currentState: Current state of the system
- context.whatAlreadyTried: Previous attempts (avoid redundancy)
- constraints: DO_NOT and MUST_VERIFY rules

### Produces
- Modified files with syntax validated
- Test results
- Any errors encountered
- What was changed

## Handoff Protocol

When complete, must invoke next agent with:
1. Full summary of what was done
2. Test results
3. Any issues that need attention
4. Explicit next action

## Constraints (Always Enforced)

- NEVER modify .git directory
- NEVER use backup files
- ALWAYS run syntax check before reporting complete
- NEVER leave console.log statements in production code

Step 3: Update Each Agent's BOOT.md or System Prompt

Add this section to each agent's bootstrap file:

# EXPLICIT CONTEXT PROTOCOL - READ THIS

You must enforce explicit context passing in ALL your work.

## The Golden Rule

**Never assume implicit context. Always require explicit context objects.**

### When Receiving a Task

If you receive a task WITHOUT the full context object, RESPOND with:

```
I need explicit context to work effectively. Please provide:

1. TASK: What specifically should I do?
2. CONTEXT:
   - workspace: (file path)
   - relevantFiles: (files to work with)
   - currentState: (what's the current situation)
   - whatAlreadyTried: (what has been attempted)
3. CONSTRAINTS:
   - DO_NOT: (what NOT to do)
   - MUST_VERIFY: (what to check)
4. ON_COMPLETE:
   - action: (what happens next)
   - successCriteria: (how to measure success)
```

### When Handing Off to Another Agent

NEVER just say "when done, continue" or "let me know when finished".

You MUST provide:
- Summary of what you accomplished
- Current state of the system
- What still needs doing
- Explicit handoff with full context

## Example Correct Handoff

```
I'm handing off to code-review agent with the following context:

TASK: Review authentication implementation

CONTEXT:
- workspace: /root/.openclaw/workspace-code-agent
- relevantFiles: ["src/auth.ts", "tests/auth.test.ts"]
- currentState: All 5 auth tests now passing
- whatAlreadyTried: ["Fixed token undefined by adding null check at line 42"]

CONSTRAINTS:
- DO_NOT: ["Don't modify the code", "Don't run git commands"]
- MUST_VERIFY: ["Verify syntax with node --check", "Verify tests pass"]

ON_COMPLETE:
- action: handoff
- nextAgent: main
- successCriteria: ["Code is clean", "No syntax errors", "Tests pass"]
```

Step 4: Create a Context Validation Hook (Optional but Recommended)

This hook validates that spawned agents receive proper context:

// Location: ~/.openclaw/hooks/context-validator/HOOK.md
---
name: context-validator
description: Validates agent spawns include proper context
emoji: "✅"
events: ["agent:spawn"]
metadata:
  openclaw:
    events: ["agent:spawn"]
---

# Context Validator Hook

Ensures every agent spawn includes required context fields.

## What It Checks

1. Task is not empty
2. Context object exists
3. Workspace is specified
4. Constraints are defined
5. onComplete action is specified

## Response

If context is incomplete, the hook returns:
```json
{
  "allowed": false,
  "error": "Missing required context fields: constraints.DO_NOT, onComplete.action"
}
```

// Location: ~/.openclaw/hooks/context-validator/handler.ts

export default async function validateContext(context: HookContext) {
  const { agentId, sessionId, payload } = context;
  
  // Extract the message or spawn payload
  const message = payload?.message || payload;
  
  // Check for explicit context structure
  const requiredFields = ['task', 'context', 'constraints', 'onComplete'];
  const missing: string[] = [];
  
  for (const field of requiredFields) {
    if (!message[field]) {
      missing.push(field);
    }
  }
  
  if (missing.length > 0) {
    return {
      allowed: false,
      error: `Missing required context fields: ${missing.join(', ')}`,
      suggestion: 'Include explicit context with task, context, constraints, and onComplete'
    };
  }
  
  // Validate context subfields
  const contextFields = ['workspace', 'relevantFiles', 'currentState'];
  const missingContext: string[] = [];
  
  for (const field of contextFields) {
    if (!message.context[field]) {
      missingContext.push(field);
    }
  }
  
  if (missingContext.length > 0) {
    return {
      allowed: false,
      error: `Missing context subfields: ${missingContext.join(', ')}`,
      suggestion: 'Provide workspace, relevantFiles, and currentState in context'
    };
  }
  
  return { allowed: true };
}
